Portfolio Case Study

Dr. Sadaf Ijaz MD Website

A responsive static website for a physician specializing in psychiatric care, women's mental health, reproductive psychiatry, medication management, and virtual care.

View Live Site Back to Portfolio
Role Frontend, UX, Accessibility
Stack HTML5, CSS3, JavaScript
Output Client-ready static deliverable

Overview

Built for clarity, trust, and handoff.

The project presents credentials, care areas, booking paths, contact details, and professional links in a calm single-page experience that is easy for patients and referring audiences to scan.

Frontend Implementation

Built the client-facing pages with semantic HTML, responsive CSS, vanilla JavaScript, and static assets without a framework or build step.

UX Structure

Organized content around visitor intent: who the physician is, what she treats, how care works, and how to book or get in touch.

Client Handoff

Kept the client site self-contained with documentation for content updates, launch review, deployment, and long-term maintenance.

Notable Decisions

Small choices that support the whole site.

  • Kept the client site as a static deliverable so it can be hosted widely and maintained easily.
  • Kept a SPA-style homepage for the primary patient experience, with standalone landing pages for About, Services, Resources, Contact, and Privacy Policy discovery.
  • Split JavaScript into small focused modules for navigation, motion preferences, video controls, contact behavior, and dynamic experience-year copy.
  • Placed trust signals near the physician bio and credentials while keeping the page readable and warm.
  • Included reduced-motion behavior and controls for the background video experience.

Employer Talking Points

What the work demonstrates.

Production Mindset

Delivered a client-ready website rather than only a visual mockup, including local documentation and handoff structure.

Healthcare Design
